# AI 사주 분석 애플리케이션 Blueprint

## 1. 개요

이 문서는 AI를 활용한 사주 분석 애플리케이션의 설계 및 개발 계획을 정의합니다. 사용자는 자신의 생년월일시와 성별을 입력하여 AI가 생성한 사주 분석 결과를 받아볼 수 있습니다. 이 애플리케이션은 React 기반으로 구축되며, 최신 웹 기술과 디자인 트렌드를 따릅니다.

## 2. 구현된 기능 및 디자인

### 2.1. 핵심 기능
- **사주 정보 입력:** 사용자는 이름, 생년월일시, 성별을 입력할 수 있는 폼을 통해 자신의 정보를 제출합니다.
- **사주 결과 표시:** 입력된 정보를 바탕으로 서버에서 사주 분석 결과를 받아와 사용자에게 보여줍니다.
- **라우팅:** `react-router-dom`을 사용하여 정보 입력 페이지와 결과 페이지 간의 전환을 관리합니다.

### 2.2. 기술 스택
- **프레임워크:** React (Vite)
- **UI 라이브러리:** Material-UI (MUI)
- **날짜/시간 선택:** `@mui/x-date-pickers`
- **상태 관리:** React Hooks (`useState`)
- **라우팅:** `react-router-dom`

### 2.3. 디자인 시스템
- **테마:** `theme.js` 파일을 통해 일관된 색상, 글꼴, 컴포넌트 스타일을 정의합니다.
- **레이아웃:** 모든 페이지에 일관된 헤더, 푸터, 그리고 메인 콘텐츠 영역을 제공하는 `Layout.jsx` 컴포넌트를 사용합니다.
- **반응형 디자인:** 다양한 화면 크기(모바일, 데스크톱)에 대응할 수 있는 반응형 레이아웃을 적용합니다.
- **컴포넌트:** MUI에서 제공하는 `Container`, `Paper`, `TextField`, `Button`, `DateTimePicker` 등의 컴포넌트를 활용하여 사용자 인터페이스를 구축합니다.

## 3. 현재 문제 및 해결 계획

### 3.1. 문제 정의
현재 애플리케이션은 정보 입력 폼(`SajuForm.jsx`)이 화면에 정상적으로 렌더링되지 않는 문제를 겪고 있습니다. 이는 `App.jsx`, `Layout.jsx`, 그리고 `SajuForm.jsx` 간의 컴포넌트 구조 및 라우팅 설정이 복잡하게 얽혀 발생한 것으로 분석됩니다. 여러 차례 수정을 시도했지만, 근본적인 구조 문제를 해결하지 못해 문제가 지속되고 있습니다.

### 3.2. 해결 계획
문제를 근본적으로 해결하기 위해 다음과 같은 단계별 계획을 수립합니다.

**1단계: 라우팅 및 레이아웃 구조 재설계 (진행중)**
- **`Layout.jsx` 수정:** `react-router-dom`의 `<Outlet />`을 사용하여 자식 라우트 컴포넌트가 렌더링될 위치를 명확히 지정합니다. 이를 통해 `Layout` 컴포넌트는 공통 UI(헤더, 푸터 등)만을 책임지게 합니다.
- **`App.jsx` 수정:** 라우팅 구조를 `<Route element={<Layout />}>` 내에 자식 라우트들을 중첩시키는 방식으로 변경합니다. 이로써 `/` 및 `/result` 경로의 모든 페이지가 `Layout` 컴포넌트를 공유하도록 만듭니다.

**2단계: 페이지 컴포넌트 (`SajuForm.jsx`) 단순화**
- `SajuForm.jsx` 내에서 불필요한 레이아웃 관련 로직을 제거하고, `Container`, `Paper` 등 MUI 컴포넌트를 사용하여 콘텐츠 자체에만 집중하도록 코드를 수정합니다. `DateTimePicker`의 `renderInput` 속성 또한 최신 MUI 버전에 맞게 올바르게 설정합니다.

**3단계: 코드 품질 및 오류 수정**
- 전체 코드에 대해 `eslint . --fix`를 실행하여 코딩 스타일을 통일하고 잠재적인 오류를 수정합니다.
- 수정 사항을 적용한 후, Vite 개발 서버의 출력과 브라우저 콘솔을 지속적으로 모니터링하여 새로운 오류가 발생하는지 확인하고 즉시 해결합니다.

이 계획을 통해 예측 가능하고 유지보수가 용이한 애플리케이션 구조를 확립하여 현재 겪고 있는 렌더링 문제를 해결할 것입니다.
